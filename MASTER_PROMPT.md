## PART 1: THE ENTERPRISE ARCHITECTURE RULES (STRICT COMPLIANCE)
1. **No RAG on Tabular Data:** We do not use vector databases. The SQL Agent extracts data. The R-Stats Agent writes deterministic math. 
2. **Context Window Protection:** The LLM must NEVER see the raw database rows. Data is saved locally to a `.csv`, and only the `data_file_path` and `extracted_columns` are passed in the LangGraph State.
3. **Secure Polyglot Execution:** Python's `exec()` or `eval()` are strictly forbidden. R code must be executed in a strictly isolated, containerized Docker sandbox.
4. **Structured Outputs:** All LLM outputs must be forced into strict Pydantic schemas using `.with_structured_output()`.
5. **Decoupled Design (Prep for K8s):** Write all code assuming this will eventually be deployed on Kubernetes. Use environment variables (`.env`) for all hostnames, ports, and API keys. Hardcoding `localhost` inside application code is forbidden.

### Step 2: The LangGraph Agents (`app/agents/`)
1. Create `app/core/llm.py` to initialize Anthropic (Claude 3.5).
2. **SQL Agent (`sql_agent.py`):** Injects standard e-commerce Postgres schema into the prompt, writes a `SELECT` query, executes it via SQLAlchemy, saves the dataframe to `data/{task_id}.csv`, and updates the state.
3. **R-Stats Agent (`r_agent.py`):** Takes the query and `extracted_columns`. Writes an R script that handles `NA` values, runs an `lm()` model, and explicitly prints a strict JSON string to stdout containing `{"p_value": float, "ate": float}`.
4. **Execution Node (`executor.py`):** Sends the generated R script and CSV path via HTTP POST to `http://r_sandbox:8001/execute`.
5. **Evaluator Node (`evaluator.py`):** Parses the JSON. Appends `is_significant = False` to the output if `p_value > 0.05`.
6. **Reviewer Node (`reviewer.py`):** Translates the statistical output into a business narrative.

### Step 3: Orchestration & Ingress (`app/core/graph.py` & `app/worker.py`)

2. Implement conditional edges: If a node fails, increment `retry_count` and append the traceback. If `retry_count >= 3`, route to a fallback termination node.
3. Update `app/worker.py` to invoke this compiled graph using the `task_id` generated by the FastAPI ingress.
